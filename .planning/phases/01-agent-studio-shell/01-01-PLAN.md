# Plan 01: App Shell And Agent Configuration Dashboard

<objective>
Build the first working AgentBid Studio slice: a polished single-user dashboard where a user can create a bidding agent, configure campaign and safety fields, and see the new agent listed with paused status for the current browser session.
</objective>

<context>
Phase 1 is the walking skeleton for the hackathon demo. There is no existing application scaffold in the repo, so this plan must create the frontend foundation and the initial in-memory agent workflow. Prioritize a credible buy-side control-plane UI over backend depth.
</context>

<requirements_addressed>
- AGENT-01: User can create a bidding agent with a campaign/agent name and brand.
- AGENT-02: User can set a campaign goal.
- AGENT-03: User can set a daily budget cap and max CPC.
- AGENT-04: User can choose manual, assisted, or autonomous mode.
- AGENT-05: User can specify target intents and blocked categories.
- DASH-01: Dashboard lists configured agents with name, brand, and status.
</requirements_addressed>

<truths>
- The first submitted agent must appear with status `paused`.
- Agent data only needs to persist across navigation within the running session; in-memory React state is acceptable.
- The UI should look like a credible buy-side product dashboard, not a toy form demo.
- The intended visual direction is a modern LLM/SaaS dashboard: Next.js, Tailwind CSS, shadcn-style components/primitives where useful, lucide-react icons, compact cards/badges/forms, and restrained control-plane styling.
- The first screen should be the usable dashboard experience, with a left app sidebar and main workspace. Do not build a marketing landing page.
- The create/edit agent flow should keep the user in context, preferably via a right-side drawer/sheet or modal rather than a separate route.
- No real ad exchange, auth, database, payments, or external API integration is required in Phase 1.
- The implementation may choose the fastest viable frontend stack, but should keep later phases easy: live simulation, analytics, safety controls, and approval queue.
</truths>

## Task 3: Build The Create Agent Form

type: implementation
files:
- `src/components/AgentForm.tsx`
- `src/app/page.tsx`

action:

  - agent/campaign name
  - brand
  - campaign goal
  - daily budget cap
  - max CPC
  - autonomy mode: manual, assisted, autonomous
  - target intents
  - blocked categories

acceptance_criteria:
- The user can launch/use the create-agent flow without reading instructions.
- Every Phase 1 agent configuration field is captured.
- The user remains on the dashboard after opening, submitting, or dismissing the create flow.
